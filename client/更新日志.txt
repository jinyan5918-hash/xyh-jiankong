【本次更新内容】（相对更早的内部版本）
--------------------------------------------------------------------------------
1. 窗口标题显示版本号（例如：企业版抖音点赞监控客户端 v1.2.0），便于员工确认是否为最新包。
2. 打包产物 exe / 文件夹名称带版本号（如 EnterpriseDouyinClient-1.2.0），避免与旧包混淆。
3. 压缩包/文件夹内附带 build_info.txt（GitHub Actions 构建时生成），可核对 Git 提交与构建时间。
4. 监控日志时间按北京时间展示；任务列表支持右键复制链接、双击在浏览器打开（需在已登录且未锁定状态下使用）。
5. 「测试系统通知」会请求服务端向当前账号已保存的企业微信 Webhook 发送测试消息（具体以当前代码为准）。

--------------------------------------------------------------------------------
【使用说明】
--------------------------------------------------------------------------------
1. 将整个文件夹解压到任意目录，勿删除同目录下的 config.json、release_version.txt 等文件。
2. 双击「EnterpriseDouyinClient-版本号.exe」启动；登录账号与密码由管理员在后台创建/分配。
3. config.json 中的 api_base 为服务器地址（与浏览器访问后台时一致，不要带 /admin）。
4. 员工须在客户端填写或由后台预填企业微信 Webhook，并按界面提示保存；管理员后台路径：/admin。
5. 若无法登录或收不到企微消息，请把本文件中的「版本号」「更新日期」及 build_info.txt（若有）发给管理员排查。

--------------------------------------------------------------------------------
【维护备忘】每次发新版请同步修改本文件并提交 Git
--------------------------------------------------------------------------------
- 更新「更新日期」「版本号」「本次更新内容」。
- 同步修改 client/release_version.txt 与 windows_client.py 中的 CLIENT_VERSION_FALLBACK。
- 确认 GitHub Actions 与 build_enterprise_client.bat 会把本文件复制进 dist 目录（随 zip 分发）。
